# Task #m600-45 — SectionDef CTRL_HEADER 47 bytes 정합

브랜치 `local/task45`. cycle 44 fix 후 잔존하던 9 bytes 결함 (CTRL_HEADER `secd`) 해소.

## 결함 자리

[src/serializer/control.rs:240-250](../../src/serializer/control.rs#L240-L250):

```rust
// 종전 주석: "한컴 정답지는 26 byte payload + 8 byte zero = 38 byte CTRL_HEADER"
} else {
    w.write_u16(0).unwrap();             // 대표Language
    w.write_bytes(&[0u8; 8]).unwrap();   // 추가 zero 8 bytes
}
```

총 38 bytes 박음. 그러나 한컴 5.0.3.x 정답지(사업관리 참조표.hwp)는 47 bytes (= 38 + 9).

## byte 단위 비교

`examples/dump_secd_full.rs` 결과:

| offset | 원본 (47) | 수정본 (38) |
|---|---|---|
| 0-17 | ctrl_id·attr·column_spacing·tab_spacing | 동일 |
| 18-19 | page_num=2 | page_num=1 (내용 차이, 별 자료) |
| 20-37 | zero | zero (동일) |
| **38-46** | **zero (9 bytes)** | **누락** |

## Fix

zero padding 8 bytes → 17 bytes:

```rust
w.write_bytes(&[0u8; 17]).unwrap(); // 한컴 5.0.3.x 정답지 패턴
```

## 남은 자료

byte 18-19 의 page_num 차이 (원본 2 vs HWPX 경유 1) 는 *내용 차이* — HWPX → HWP IR 변환 시 page_num 자료 잘못 박는 자리. 한컴 손상 알람 원인은 아님 (size 정합만 한컴이 검사). 별 cycle 후보.
