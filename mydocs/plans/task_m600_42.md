# Task #m600-42 — server export 자리의 compressed flag 손실 fix (한컴 손상 알람 해소)

브랜치 `local/task42`. 사용자 보고 — *rhwp-studio 에서 텍스트 편집 후 hwp 다운로드 → 한컴 편집기로 열면 "손상된 문서" 알람*. studio 에서는 정상 표시.

## 진단

`/Users/yuniba_01/Downloads/icon/사업관리 참조표.hwp` (원본, 열림) 와 `사업관리 참조표_rhwp_modified.hwp` (수정본, 안 열림) 의 CFB stream 비교:

| 자료 | 원본 | 수정본 |
|---|---|---|
| FileHeader properties | `0x00000001` (compressed=true) | `0x00000000` (compressed=false) |
| FileHeader version | 5.1.0.1 | 5.1.0.0 |
| BodyText/Section0 | 6106 bytes (zlib) | 15475 bytes (원시 데이터, 2.5× 큼) |
| DocInfo | 2758 bytes (zlib) | 14618 bytes (원시, 5.3× 큼) |
| DocOptions/_LinkDoc | 524 bytes (zlib) | 524 bytes (원시 UTF-16LE) |
| PrvImage | 46271 bytes (정상 PNG) | 68 bytes (1×1 빈 PNG) |
| PrvText | 2046 bytes | 4 bytes (`\r\n`) |

수정본은 FileHeader 자체가 `compressed=false` 라 박혀 있고 BodyText/DocInfo 도 실제로 압축 안 됨. 내부 정합성은 맞지만 한컴 편집기는 `compressed=false` 자체를 손상으로 인식.

## 원인 자리

[examples/roundtrip_hwp_header.rs](../../examples/roundtrip_hwp_header.rs) 진단:

```
=== 원본 ===
  [orig] properties=0x00000001 compressed=true

=== IR (HWP parse) ===
  header.compressed=true flags=0x00000001 raw_data=Some(256)

=== serialize_document 결과 ===
  [ser] properties=0x00000001 compressed=true  ✅

=== DocumentCore.export_hwp_native ===
  [core] properties=0x00000001 compressed=true  ✅

--- HWPX 거쳐서 ---
  hwpx bytes len=15258
  re-parse IR header.compressed=false flags=0x0 raw_data=None  ❌
--- HWPX→HWP adapter 후 ---
  [final] properties=0x00000001 compressed=true  ✅
```

HWPX parser 가 FileHeader 를 default (compressed=false) 로 박는다. `export_hwp_with_adapter` 는 `convert_if_hwpx_source` 안의 `normalize_file_header_for_hwp` 로 compressed=true·flags|=0x01 박지만, **server export 자리는 `rhwp::serialize_document(doc)` 를 직접 호출** — adapter 미경유.

server 의 손상 자리 3곳 ([rhwp-server/src/main.rs](../../rhwp-server/src/main.rs)):
- `export` (line 605) — 다운로드 endpoint
- `save_document` (line 642) — 저장 endpoint
- `save_as` (line 677) — 다른 이름으로 저장

세 자리 모두 `s.core.document()` 로 immutable borrow 만 잡고 `rhwp::serialize_document(doc)` 호출.

## Fix

세 자리 전부 `s.core.export_hwp_with_adapter()` 호출로 변경. `&mut s` 로 lock 박음. adapter 는 source_format=Hwp 자리에서 no-op 이라 HWP 원본 출처에는 영향 없음. HWPX 출처 (server snapshot 자료) 에만 normalize_file_header_for_hwp 가 작동해 compressed=true·flags|=0x01·raw_data=None 박는다.
