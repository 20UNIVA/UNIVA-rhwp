## Wrapper 정합

HWPX 표준에서 wrapper 가 variant 마다 다르다:

| Variant | Wrapper | 자리 |
|---|---|---|
| Equation | `<hp:run><hp:equation .../></hp:run>` | hp:run 자식 |
| CharOverlap | `<hp:run><hp:compose></hp:compose></hp:run>` | hp:run 자식 |
| ColumnDef | `<hp:p><hp:ctrl><hp:colPr/></hp:ctrl></hp:p>` | paragraph 자식의 hp:ctrl |

CharOverlap 은 `Empty Tag <hp:compose/>` 가 parser parse_paragraph 에서 무시되므로 `Start + End` 쌍 으로 박는다 ([parser line 415-419](../../src/parser/hwpx/section.rs#L415)).

## Fix

[src/serializer/hwpx/table.rs](../../src/serializer/hwpx/table.rs) `write_sub_list`:

- *hp:run 자식 자리* 에 Equation·CharOverlap 분기 추가. Equation 은 body 자리의 `render_equation` 함수 (cycle 37 에서 `pub(super)` 로 expose) 를 raw bytes 로 inner writer 에 박는다. CharOverlap 은 빈 `<hp:compose></hp:compose>` 만 박아 한 건의 control 을 round-trip 한다.
- *hp:ctrl wrapper 자리* 에 ColumnDef 분기 추가. `<hp:ctrl><hp:colPr/></hp:ctrl>` 최소 wrapper. parser `parse_col_pr` 는 attribute 가 비어 있어도 `ColumnDef::default()` 로 한 건을 인식한다.

CharOverlap·ColumnDef 의 *정확한 attribute 복원* (circleType·composeText·열 수·간격) 은 별 cycle 로 분리. 본 cycle 의 목표는 round-trip control 개수 보존.
