## 배경

Task #m600-24 사이클 종결 후 사용자 시각 자료 자료 자료 — *새로고침 시 서식이 깨짐*. 진단 결과 결함 자리가 *paragraph end marker 흐름*이 아닌 *paragraph.text 자체에 ㅇ (U+3147) 가 박힌 자료*임을 확인.

server export hwpx 의 section0.xml byte hex 자료:
```
Reference Table\xe3\x85\x87</hp:t></hp:run>
                ^^^^^^^^^^^
                U+3147 ㅇ UTF-8 (e3 85 87)
```

- workbench reply 의 `before/after.cell.paragraphs[0].runs[0].text` = 75자 (ㅇ 없음)
- export hwpx 의 hp:t = 76자 (ㅇ 박힘)
- ir-slice 의 cell text = 75자 (ㅇ 없음)

`render_paragraph_parts → render_run_content` 자료 정독 결과 *paragraph.text 자체*를 그대로 hp:t 로 출력. 추가 1자를 박는 자리는 없음. 즉 *paragraph.text 자체에 76자가 박혔다*. ir-slice·workbench reply 는 *run 단위로 char_shapes 자르기* path 를 거쳐 75자만 반환 — *마지막 1자 ㅇ 는 char_shape 가 매핑되지 않은 자료*.

## 결함 자리 후보

- **후보 A** (parser path) — 원본 hwpx 자체 또는 server hwpx parser 가 *paragraph.text 끝에 ㅇ 추가*. HWP5/HWPX 에서 한컴 편집기가 paragraph end 자료를 hp:t 안에 ㅇ 로 직렬화한 자료라면, parser 가 그 ㅇ 를 그대로 paragraph.text 로 가져옴.
- **후보 B** (replace_cell_runs chain) — `delete_text_in_cell_native(0, para_len)` 가 *마지막 1자를 남기는* 자료. `para_len = text.chars().count()` 산출 자리에서 *paragraph end marker 1자 자료가 char_count 에는 있지만 text.chars() 에는 없어 산출값이 부족*. 결과적으로 delete 가 부분만 일어남.

후보 A 라면 *parser 결함*. 패치 자리는 `src/parser/hwpx/section.rs:583` 근방. 후보 B 라면 *replace_cell_runs_native 의 para_len 산출* 또는 *delete_text_in_cell_native count 계산*.

## Stage 2 — 패치 자리 (Stage 1 결과로 결정)

### 후보 A 인 경우 — parser 자리

[src/parser/hwpx/section.rs:583](../../src/parser/hwpx/section.rs#L583) 의 `char_count = utf16_pos + 1` 자리 부근에서 *paragraph.text 자료에 ㅇ 가 박히는 경로* 식별. parser 가 hp:t 의 마지막 char (U+3147) 를 *paragraph end marker* 로 인식해 text 에서 제외해야 하는데 그대로 박는 자료.

패치 — parser 자리에서 *paragraph.text 끝의 paragraph end marker (U+3147 또는 0x000D) 자료 제거*.

### 후보 B 인 경우 — replace_cell_runs chain

[src/document_core/commands/text_editing.rs:849](../../src/document_core/commands/text_editing.rs#L849) `let para_len = cell_para_ref.text.chars().count();` 자리. `para.char_count - 1` (UTF-16 unit) vs `text.chars().count()` 차이 자료. paragraph end marker 가 *char_count 에만 있고 text 에는 없는* 규약 정합이면 산출값이 맞아야 하는데 ㅇ 가 남는다는 자료는 *paragraph.text 에 ㅇ 자료 자체가 박힌 자료* — 결국 후보 A 로 귀착할 가능성.
