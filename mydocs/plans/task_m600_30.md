## Fix

`write_sub_list` 안 hp:run 박는 자리:

```rust
let text_u16: Vec<u16> = para.text.encode_utf16().collect();
let css = &para.char_shapes;
if css.is_empty() || text_u16.is_empty() {
    // empty text 자체 자체 자체 자체 자체 hp:run 박지 못하면 char_shape_id 자료 손실.
    let cs = css.first().map(|r| r.char_shape_id).unwrap_or(0);
    let cs_str = cs.to_string();
    start_tag_attrs(w, "hp:run", &[("charPrIDRef", &cs_str)])?;
    write_cell_text(w, &para.text)?;
    end_tag(w, "hp:run")?;
} else {
    for (i, cs_ref) in css.iter().enumerate() {
        let start = (cs_ref.start_pos as usize).min(text_u16.len());
        let end = css.get(i + 1)
            .map(|c| c.start_pos as usize)
            .unwrap_or(text_u16.len())
            .min(text_u16.len());
        if end <= start { continue; }
        let segment = String::from_utf16_lossy(&text_u16[start..end]);
        let cs_str = cs_ref.char_shape_id.to_string();
        start_tag_attrs(w, "hp:run", &[("charPrIDRef", &cs_str)])?;
        write_cell_text(w, &segment)?;
        end_tag(w, "hp:run")?;
    }
}
```

핵심 자료:
- `char_shapes` 별로 별도 `<hp:run>` 박음 — start_pos 자료 (utf16 offset) 따라 paragraph.text 분할.
- `start_pos` 자료가 paragraph end marker·control 자체 자체 포함된 utf16 위치라 `text_u16.len()` 자체 자체 clamp.
- empty text 자체 자체 자체 (`text.is_empty()`) 자체 자체 빈 `<hp:run charPrIDRef=cs>` 자체 박아 char_shape_id 자료 보존.
