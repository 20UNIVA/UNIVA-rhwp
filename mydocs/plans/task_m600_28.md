## Fix — 4개 결함 자료

### Fix 1 — `serialize_table` raw_ctrl_data 빈 자료 fallback

[src/serializer/control.rs:455-466](../../src/serializer/control.rs#L455-L466) 의 `serialize_table` 에 *raw_ctrl_data 가 비어있으면 common 으로부터 합성* fallback 추가.

```rust
fn serialize_table(table: &Table, level: u16, records: &mut Vec<Record>) {
    let synthesized;
    let ctrl_data: &[u8] = if !table.raw_ctrl_data.is_empty() {
        &table.raw_ctrl_data
    } else {
        synthesized = build_table_ctrl_data_from_common(table);
        &synthesized
    };
    records.push(make_ctrl_record(tags::CTRL_TABLE, level, ctrl_data));
    // ... 이하 동일
}

/// HWPX parser 자료 입력 등 raw_ctrl_data 가 비어있는 경우 common 으로부터 합성.
fn build_table_ctrl_data_from_common(table: &Table) -> Vec<u8> {
    let mut data = serialize_common_obj_attr(&table.common);
    // HWPX parser 가 common.margin 자료 안 채우고 table.outer_margin_* 만 채움 →
    // serialize_common_obj_attr 의 margin 자리 (offset 24..32) 를 outer_margin_* 로 덮어쓴다.
    if data.len() >= 32 {
        data[24..26].copy_from_slice(&table.outer_margin_left.to_le_bytes());
        data[26..28].copy_from_slice(&table.outer_margin_right.to_le_bytes());
        data[28..30].copy_from_slice(&table.outer_margin_top.to_le_bytes());
        data[30..32].copy_from_slice(&table.outer_margin_bottom.to_le_bytes());
    }
    data
}
```

기존 `convert_hwpx_to_hwp_ir` (document_core/converters/hwpx_to_hwp.rs:1064-1080) 의 합성 로직과 동등하나, *production export path 가 이 변환을 호출하지 않으므로* serializer 안에서 직접 fallback 박는다.

### Fix 2 — HWPX `table_page_break_str` 매핑 역방향

[src/serializer/hwpx/table.rs:438-444](../../src/serializer/hwpx/table.rs#L438-L444) — HWPX parser (section.rs:1402-1404) 의 주석 정합:

```
HWPX "CELL"  ↔ HWP5 RowBreak  (한컴 명명 — 셀 안 내부 분할 허용)
HWPX "TABLE" ↔ HWP5 CellBreak (한컴 명명 — 셀 단위로 표를 나눔)
```

종전 serializer 만 반대로 박혀 round-trip 시 RowBreak ↔ CellBreak 자료가 서로 바뀜. 사용자 시각상 *2페이지 표가 칸별로 찢어짐* 증상의 직접 원인.

```rust
match pb {
    None => "NONE",
    RowBreak => "CELL",       // 종전 "TABLE"
    CellBreak => "TABLE",     // 종전 "CELL"
}
```

### Fix 3 — cell paragraph controls 박지 않음 (nested table 손실)

[src/serializer/hwpx/table.rs:278](../../src/serializer/hwpx/table.rs#L278) — `write_sub_list` 안 cell paragraph 의 *text 만* 박고 *controls 무시*. *3페이지 이중 표의 내부 표가 사라짐* 증상의 직접 원인.

```rust
write_cell_text(w, &para.text)?;
// 추가 자료 — cell paragraph 의 controls (nested table·picture) 박음.
for ctrl in &para.controls {
    match ctrl {
        Control::Table(t) => write_table(w, t, ctx)?,
        Control::Picture(pic) => crate::serializer::hwpx::picture::write_picture(w, pic, ctx)?,
        _ => {}
    }
}
end_tag(w, "hp:run")?;
```

### Fix 4 — HWPX template pagePr/margin 고정값 (PageDef 손실)

[src/serializer/hwpx/section.rs](../../src/serializer/hwpx/section.rs) — `EMPTY_SECTION_XML` template 의 `<hp:pagePr ...><hp:margin .../></hp:pagePr>` 영역이 *고정값* (margin L=8504 R=8504 등) 박혀있고 PageDef 자료로 덮어쓰지 않음. *오른쪽 여백 줄어듦* 증상의 직접 원인 (사용자 시각).

`replace_page_pr` 헬퍼 추가 — `write_section` 안에서 PageDef 자료로 `<hp:pagePr>` 영역 교체.

```rust
out = replace_page_pr(&out, &section.section_def.page_def);
```

ir-diff 의 비교 범위가 PageDef 를 포함하지 않아 자동 검출 안 됨. 사용자 시각 보고로만 catch.
