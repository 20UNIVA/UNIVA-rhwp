# Task #m600-43 — HWPX serializer 가 원본 Preview 보존 (PrvImage·PrvText 손실 해소)

브랜치 `local/task43`. cycle 42 에서 한컴 손상 알람 직접 원인 (compressed flag) 해소 후, 부수 결함인 *원본 PrvImage·PrvText 손실* 정리.

## 결함 자리

[src/serializer/hwpx/static_assets.rs:56-66](../../src/serializer/hwpx/static_assets.rs#L56-L66):

```rust
pub const PRV_TEXT: &[u8] = b"\r\n";
pub const PRV_IMAGE_PNG: &[u8] = &[0x89, 0x50, ..., 0x82];  // 1×1 투명 PNG
```

[src/serializer/hwpx/mod.rs:67-69](../../src/serializer/hwpx/mod.rs#L67-L69) (cycle 43 이전):

```rust
z.write_deflated("Preview/PrvText.txt", PRV_TEXT)?;
z.write_deflated("Preview/PrvImage.png", PRV_IMAGE_PNG)?;
```

HWPX serializer 는 *항상* default 자료로 박는다 — 원본 `doc.preview` 의 image·text 가 무시됨. HWP → HWPX → HWP round-trip 시:
- PrvImage 46271 → 68 (1×1 빈 PNG)
- PrvText 2046 → 4 (`\r\n`)

한컴 편집기의 파일 목록 썸네일·문서 미리보기 기능에 영향. 사용자 보고의 "손상 알람" 직접 원인은 아니지만 (cycle 42 자리), 사용자 시각에는 *파일이 비어 있는 듯한* 인상.

## Wrapper 정합 점검

HWPX 의 `Preview/PrvText.txt` 는 **UTF-8** (HWP 의 `/PrvText` 는 UTF-16LE). HWPX parser [src/parser/hwpx/contract_streams.rs:70-74](../../src/parser/hwpx/contract_streams.rs#L70-L74) 가 *UTF-8 → encode_utf16 → bytes* 변환해서 `/PrvText` 박는다. 즉 serializer 도 *UTF-8 raw bytes* 박아야 contract 정합.

PrvImage 는 PNG passthrough — HWP 의 `/PrvImage` 와 byte-identical.

## Fix

[src/serializer/hwpx/mod.rs](../../src/serializer/hwpx/mod.rs) 의 Preview 박는 자리:

```rust
let prv_text_bytes = doc.preview
    .as_ref()
    .and_then(|p| p.text.as_ref())
    .map(|t| t.as_bytes().to_vec())  // UTF-8 raw
    .unwrap_or_else(|| PRV_TEXT.to_vec());
let prv_image_bytes = doc.preview
    .as_ref()
    .and_then(|p| p.image.as_ref())
    .map(|img| img.data.as_slice())
    .unwrap_or(PRV_IMAGE_PNG);
z.write_deflated("Preview/PrvText.txt", &prv_text_bytes)?;
z.write_deflated("Preview/PrvImage.png", prv_image_bytes)?;
```

IR 에 preview 있으면 그대로 박고, 없으면 default fallback. 빈 문서·`Document::default()` 호환.
