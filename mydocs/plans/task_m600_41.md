## 분석 — sub-agent 오판

sub-agent 는 cycle 33 의 serializer 코드만 보고 "fieldBegin 직후 바로 fieldEnd 라 본문이 빈 자리" 로 추정했다. 그러나 실제 동작은 *visible text 가 paragraph.text 경로로 별도 박힌 자료* — `write_cell_text(w, &para.text)` 가 `<hp:run><hp:t>...</hp:t></hp:run>` 으로 박고, parser 의 `parse_paragraph` 가 그 text 를 그대로 paragraph.text 로 다시 채운다.

cycle 33 의 `<hp:ctrl><hp:fieldBegin/><hp:fieldEnd/></hp:ctrl>` 는 *Field 메타 정보 (field_id·type·command)* 만 박는 자리. visible text 보존은 paragraph.text 경로가 책임진다 — 두 경로가 분리되어 있어 cycle 33 wrapper 가 비어 있어도 사용자가 화면에서 보는 글자는 무손실.
