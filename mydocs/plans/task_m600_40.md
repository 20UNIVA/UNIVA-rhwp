# Task #m600-40 — cell CharOverlap attribute 정밀 복원

브랜치 `local/task40` (cycle 39 분석에서 *high 위험* 으로 식별된 자리). cycle 37 의 minimal wrapper (`<hp:compose></hp:compose>`) 가 round-trip 후 *겹쳐질 글자 자체* 를 default 로 덮어쓰는 자리 해소.

## 결함 자리

[src/serializer/hwpx/table.rs:328-336 (cycle 37 자리)](../../src/serializer/hwpx/table.rs) — 종전:

```rust
Control::CharOverlap(_) => {
    start_tag(w, "hp:compose")?;
    end_tag(w, "hp:compose")?;
}
```

parser [parse_compose](../../src/parser/hwpx/section.rs#L4222-L4294) 가 읽는 자료는 4개:

| HWPX attribute / element | IR 필드 | 의미 |
|---|---|---|
| `circleType` | `border_type` (u8) | CHAR/SHAPE_CIRCLE/.../SHAPE_REVERSAL_TIRANGLE |
| `charSz` | `inner_char_size` (i8) | 내부 글자 크기 (%) |
| `composeType` | `expansion` (u8) | OVERLAP=1, SPREAD=0 |
| `<hp:composeText>` 자식 | `chars` (Vec<char>) | 겹쳐질 글자 |
| `<hp:charPr prIDRef>` 자식 | `char_shape_ids` (Vec<u32>) | 각 글자의 char_shape |

cycle 37 은 *control 개수* 만 round-trip — 그 5개 자료가 default (CHAR·0·SPREAD·빈 chars·빈 char_shape_ids).

## Fix

[src/serializer/hwpx/table.rs](../../src/serializer/hwpx/table.rs) 의 CharOverlap 분기:

1. 3개 attribute 매핑 — `border_type` ↔ `circleType` (역방향 7-way 매핑), `inner_char_size` ↔ `charSz`, `expansion` ↔ `composeType`.
2. `chars` 가 비어 있지 않으면 `<hp:composeText>` 자식 element 박음 (quick_xml `BytesText` 로 raw 글자 박기).
3. 각 `char_shape_ids` 마다 `<hp:charPr prIDRef="N"/>` Empty Tag 박음.
