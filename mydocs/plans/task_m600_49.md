# Task #m600-49 — normalize_paragraph_char_border_fills 비활성화 (페이지 배경색 확산 결함 해소)

브랜치 `local/task49`. 사용자 보고 — 사업관리 참조표 문서의 4행 3열 cell 배경을 분홍으로 변경 후 한컴에서 열면 *페이지 전체가 분홍색*. rhwp-studio 에서는 정상.

## 결함 자리

[src/document_core/converters/hwpx_to_hwp.rs:356](../../src/document_core/converters/hwpx_to_hwp.rs#L356) `normalize_paragraph_char_border_fills`:

```rust
if is_transparent_paragraph_no_fill_candidate(border_fill) {
    border_fill.fill.fill_type = FillType::None;
    // border_fill 자체 자체 None 박음
}
```

조건 (`is_transparent_paragraph_no_fill_candidate`):
- 모든 border = None
- fill_type = Solid
- alpha == 0 && background_color == 0xffffffff

paragraph·char_shape 만 참조하는 border_fill 중 위 조건 충족 시 None 박는 자료. 원래 의도는 *transparent white paragraph background 정합*.

그러나 한컴이 박은 원본 hwp (사업관리 참조표.hwp) 의 bf[1] / bf[2] / bf[8] 자료가:
- `Solid(background=0xffffffff, pattern_color=0x999999/0xff000000/0x000000, pattern_type=-1)`

위 normalize 조건 충족 → None 박힘. 그러나 *원본 한컴은 이 Solid white 자료를 page background 등의 자료로 사용*.

## 사용자 시나리오

1. 원본 hwp upload → IR.bf[1]=Solid(white) 등 (정상)
2. HWPX export → HWPX 박힘
3. HWPX → re-parse → IR
4. `convert_if_hwpx_source` → `normalize_paragraph_char_border_fills` → **bf[1]·bf[2]·bf[8] None 박힘**
5. HWP serialize → 새 hwp
6. 사용자가 4행 3열 cell 분홍색 변경 (cell.border_fill_id=17 박힘, bf[16]=Solid(분홍) 추가)
7. 다운로드 → 한컴이 *bf[1]=None 자료 자체 자체 page background 자료 누락 → bf[16] 분홍 색이 페이지로 확산*

## byte 단위 진단 (`dump_para_border_fills`)

| index | 원본 사업관리 | pink_cell 자체 |
|---|---|---|
| bf[1] | Solid(white, pattern=999) | **None** ❌ |
| bf[2] | Solid(white, pattern=ff000000) | **None** ❌ |
| bf[8] | Solid(white, pattern=000000) | **None** ❌ |
| bf[16] | (없음) | Solid(0x00dcd1ff 분홍) — 사용자 추가 |

## Fix

`normalize_paragraph_char_border_fills` 자체 비활성화. 원본 IR 자체 자체 박지 않음.

```rust
fn normalize_paragraph_char_border_fills(doc: &mut Document, report: &mut AdapterReport) {
    // Task #m600-49 — 비활성화. 원본 한컴이 박은 Solid white border_fill 자료가
    // page background 등의 자료로 사용되므로 None 박지 않는다.
    let _ = doc;
    let _ = report;
}
```

향후 paragraph background 정합 자료가 정확히 필요한 자리가 식별되면 더 좁은 조건으로 재도입.
