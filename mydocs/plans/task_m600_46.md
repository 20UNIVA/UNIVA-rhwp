## 결함 자리

[src/serializer/hwpx/header.rs:553](../../src/serializer/hwpx/header.rs#L553):

// 종전: shadow_type != 0 인 경우만 박음

원본 사업관리 참조표.hwp 의 char_shapes[0] 은 *shadow_type=0 (NONE)* 이지만 *shadow_offset_x=10, shadow_offset_y=10, shadow_color=0x00b2b2b2* 값이 박혀 있음. serializer 가 NONE 일 때 `hh:shadow` 자체를 박지 않아 HWPX 거친 IR 에서 shadow 자료가 0 으로 떨어짐.

parser [src/parser/hwpx/header.rs:712-714](../../src/parser/hwpx/header.rs#L712-L714) 는 type=NONE 인 경우에도 offsetX·offsetY·color 를 읽는 자료 ([test_parse_char_pr_preserves_shadow_offsets_even_when_shadow_is_none 자료](../../src/parser/hwpx/header.rs#L2182)). 즉 *parser-serializer 비대칭*.

## Fix

shadow 박는 조건을 *type 외에 offset·color 값도 검사*:

```rust
if cs.shadow_type != 0
    || cs.shadow_offset_x != 0
    || cs.shadow_offset_y != 0
    || cs.shadow_color != 0
{
    let shadow_type_str = if cs.shadow_type != 0 { "CONTINUOUS" } else { "NONE" };
    empty_tag(w, "hh:shadow",
        &[("type", shadow_type_str), ("color", &color_hex(cs.shadow_color)),
          ("offsetX", &cs.shadow_offset_x.to_string()),
          ("offsetY", &cs.shadow_offset_y.to_string())])?;
}
```
