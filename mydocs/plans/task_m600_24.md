# Task #m600-24 — `replace_cell_runs` 직후 cell.height 폭증·그라데이션 손실 fix (수행+구현 통합 계획서)

## 배경

작업 루트 [docs/24-cell-paginate-blowup-spec.md](../../../docs/24-cell-paginate-blowup-spec.md) 가 진단 자료를 담는다. 핵심 자리만 옮겨 놓는다.

- `replace_cell_runs` 호출 후 server in-memory IR 의 cell.height·border_fill_id·line_segs lh 가 모두 무결한데, *클라 WASM paginate* 가 *변경 안 한 행 (row 0, row 2)* 의 cell.height 를 380 → 508·2639 로 폭증시킴. 사용자는 *그라데이션 색띠가 사라진 듯한* 시각 손실을 본다 (얇은 색띠가 큼지막한 색 덩어리로 부풀어 오른 부수효과).
- 의심 1순위 — [src/renderer/composer/line_breaking.rs:922-947](../../src/renderer/composer/line_breaking.rs#L922-L947) 의 `make_line_seg` 안 `max_font_size > 0 ? max_font_size : 12.0` 폴백. 빈 paragraph 자리 (line 945-947) 가 `make_line_seg(0, 0.0)` 으로 호출되어 *12pt 폴백 적용 → line_height_hwp 큼*. 원본 hwpx 의 빈 셀 paragraph 는 *font_size 1.0pt 자리 (lh=1000)* 인데 폴백이 적용되면 12pt 자리 lh 가 박혀 cell.height 폭증.

## 해결 방향 (spec §5 fix 후보 A)

빈 paragraph 자리에서 *para.char_shapes 의 첫 항목* 으로부터 font_size 를 끌어와 `make_line_seg(0, fs_first)` 호출. char_shapes 자료가 비어 있을 때만 12.0 폴백 유지. 가장 좁은 패치 폭.

## Stage 2 — 후보 A 패치

### before ([line_breaking.rs:945-947](../../src/renderer/composer/line_breaking.rs#L945-L947))

```rust
    if para.text.is_empty() {
        para.line_segs = vec![make_line_seg(0, 0.0)];
        return;
    }
```

`make_line_seg(0, 0.0)` 호출 → 함수 안에서 `max_font_size=0.0` → `fs=12.0` 폴백 → 12pt 자리 line_height 산출 → cell.height 폭증.

### after

```rust
    if para.text.is_empty() {
        // char_shapes 첫 항목의 font_size 차용. 빈 paragraph 라도 한컴 IR 규약상
        // char_shape ref 한 개는 반드시 박혀 있다. 자료가 살아 있으면 그 자료로
        // line_height 산출, 비어 있을 때만 make_line_seg 내부 12.0 폴백 적용.
        let fs_first = para
            .char_shapes
            .first()
            .and_then(|cs| styles.char_styles.get(cs.style_id as usize))
            .map(|s| s.font_size)
            .unwrap_or(0.0);
        para.line_segs = vec![make_line_seg(0, fs_first)];
        return;
    }
```

(주의 — `cs.style_id` 의 정확한 필드명·타입은 Stage 1 자료 확보 후 확정. `CharShapeRef` 정의를 그대로 옮긴다.)

### fill_lines 결과 자리 추가 패치

Stage 1 자료에서 *fill_lines 결과 행 의 max_font_size=0* 자리가 확인되면 같은 자리에 fallback 추가. 자리 미확인 시 Stage 2 자체에서 추가 패치는 보류.

```rust
#[test]
fn empty_paragraph_uses_first_char_shape_font_size() {
    // 빈 텍스트 + char_shapes[0] font_size=1.0pt → lh=1000 (HWPUNIT)
    // 폴백 12pt 자리 lh ≈ 1200 와 명확히 구분되어야 함.
    let mut para = Paragraph {
        text: String::new(),
        char_shapes: vec![CharShapeRef { utf16_start: 0, style_id: 0 }],
        ..Default::default()
    };
    let styles = test_styles_with_font_size(0, 1.0);
    reflow_line_segs(&mut para, 400.0, &styles, 96.0);
    assert_eq!(para.line_segs.len(), 1);
    assert!(
        para.line_segs[0].line_height < 200,
        "fs=1pt 자리 line_height 가 12pt 폴백 자리 (≈1200) 와 명확히 구분되어야 함. got={}",
        para.line_segs[0].line_height
    );
}

#[test]
fn empty_paragraph_with_no_char_shapes_falls_back_to_12pt() {
    // char_shapes 비어 있을 때만 12pt 폴백 동작.
    let mut para = Paragraph {
        text: String::new(),
        char_shapes: vec![],
        ..Default::default()
    };
    let styles = test_styles_empty();
    reflow_line_segs(&mut para, 400.0, &styles, 96.0);
    assert_eq!(para.line_segs.len(), 1);
    assert!(para.line_segs[0].line_height > 1000);
}
```

(`CharShapeRef`·`test_styles_with_font_size`·`test_styles_empty` 의 정확한 구성은 Stage 1 자료 확보 후 확정.)
