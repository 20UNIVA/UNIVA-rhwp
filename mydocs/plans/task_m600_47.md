# Task #m600-47 — HWPX lineSpacing 박을 때 line_spacing_v2 우선

브랜치 `local/task47`. 사용자 보고 — 애국가.hwp 의 paragraph 가 한컴에서 모두 같은 자리에 겹쳐 표시. rhwp-studio 에서는 정상.

## 결함 자리

[src/serializer/hwpx/header.rs:911](../../src/serializer/hwpx/header.rs#L911):

```rust
("value", &ps.line_spacing.to_string()),
```

ParaShape 의 두 line_spacing 필드:
- `line_spacing` (legacy, HWP 5.0.2.4 이하)
- `line_spacing_v2` (5.0.2.5+, 한컴 5.0.3.x 정답지 사용)

한컴이 박은 원본 hwp 의 ps[20] 은 `line_spacing=2, line_spacing_v2=160`. legacy 필드가 2 로 박혀 있고 v2 필드만 정상값 160. 한컴 편집기는 v2 우선 적용.

rhwp HWPX serializer 는 legacy `ps.line_spacing` (=2) 박음 → HWPX re-parse 시 IR.line_spacing=2 → HWP serialize 시 byte offset 24-27 에 2 박힘 → 한컴이 *줄 간격 2 (거의 0)* 로 해석 → paragraph 겹침.

## Fix

```rust
let line_spacing_value = if ps.line_spacing_v2 != 0 {
    ps.line_spacing_v2 as i32
} else {
    ps.line_spacing
};
("value", &line_spacing_value.to_string()),
```

`line_spacing_v2` 가 0 이 아니면 우선, 0 이면 legacy fallback.
